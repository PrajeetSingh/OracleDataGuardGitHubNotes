## Configure Network: Add entries to TNSNames.ora and Listener.ora

Notice the use of the `SID`, rather than the `SERVICE_NAME` in the entries. This is important as the broker will need to connect to the databases when they are down, so the services will not be present.

ps -ef | grep smon
oracle    5736     1  0 Feb15 ?        00:00:00 ora_smon_cdbapp1

*Here scd1 is Primary and mcd1 is Standby.*

scd1 =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = scd1.localdomain)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SID = cdbapp1)
    )
  )

mcd1 =
  (DESCRIPTION =
    (ADDRESS_LIST =
      (ADDRESS = (PROTOCOL = TCP)(HOST = mcd1.localdomain)(PORT = 1521))
    )
    (CONNECT_DATA =
      (SID = cdbapp1)
    )
  )
  
Since the broker will need to connect to the database when it's down, we can't rely on auto-registration with the listener, hence the explicit entry for the database.

SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (ORACLE_HOME = /u01/app/oracle/product/19.0.0/dbhome_1)
      (SID_NAME = cdbapp1)
      (ENVS="TNS_ADMIN=/u01/app/oracle/product/19.0.0/dbhome_1/network/admin")
    )
  )

## Other Init Parameters

-- Enables or disables the sending of redo logs to remote destinations and specifies the DB_UNIQUE_NAME for all databases in the Data Guard configuration.

SQL> show parameter log_archive_config
ALTER SYSTEM SET log_archive_config='dg_config=(cdbapp1_sec,cdbapp1_mcd)' SCOPE=both;

SQL> show parameter log_archive_dest_1
alter system set log_archive_dest_1 = 'LOCATION=USE_DB_RECOVERY_FILE_DEST  VALID_FOR=(ALL_LOGFILES,ALL_ROLES) DB_UNIQUE_NAME=cdbapp1_sec' scope=both;

-- log_archive_dest_2 will be set by Data Guard Broker, so set it to Null
SQL> show parameter log_archive_dest_2
alter system set log_archive_dest_2='' scope=both;

Specifies the state (ENABLE or DEFER) of the corresponding LOG_ARCHIVE_DEST_n parameter.

LOG_ARCHIVE_DEST_1 archives redo data generated by the primary database from the local online redo log files to the local archived redo log files directory.

LOG_ARCHIVE_DEST_2 is valid only for the primary role. This destination transmits redo data to the remote physical standby destination directory.

SQL> show parameter log_archive_dest_state_1
